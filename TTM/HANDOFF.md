# TTM316 — Text Token Manager: Developer Handoff

## Overview

TTM316 is a Figma plugin and companion web app for managing a complete typography token system. It mirrors the architecture of CTM316 (Color Token Manager) but outputs **FLOAT variables** for font sizes and **text styles** instead of color variables and paint styles.

**Two deliverables share the same logic and UI:**
- `Figma_Plugin/` — runs inside Figma, writes variables and text styles to the document
- `Web_App/` — standalone browser app, same UI without Figma API, exports CSS/SCSS/JSON

## Core Concepts

### Type Scale
A numeric progression of font sizes generated by one of three methods:

| Method   | Algorithm |
|----------|-----------|
| Modular  | `seed × ratio^i` for i = 0..steps-1 |
| Linear   | `min + (i/(steps-1)) × (max-min)`, evenly spaced |
| Custom   | User-supplied comma-separated px values |

Each step has an `index` (0-based), a `name` (from naming scheme), a `px` value, and a `rem` value.

**Per-step overrides:** The user can manually override any individual step's px value directly in the scale preview bar. Stored in `scale.scaleOverrides` as `{ "index": px }`.

**Naming schemes:** `numeric` (1,2,3), `sizeLabels` (xs,s,m,l…), `rem` (0.75rem…), `px` (12px…), `custom` (user CSV).

### Roles
A role is a named text style group (e.g. "Heading", "Body", "Caption"). Each role has:
- A **font slot** (which font family to use)
- A **text transform** (none / uppercase / lowercase / capitalize)
- One or more **variations** — the individual named text styles within the role

### Variations
Each variation maps to one text style per font weight. A variation has:
- `name` — the variation identifier (e.g. "h1", "large", "base")
- `scaleIndex` — which step in the type scale to use for font size
- `lineHeight` — `{ unit: "PERCENT"|"PIXELS"|"AUTO", value: number }`
- `letterSpacing` — `{ unit: "PERCENT"|"PIXELS", value: number }`

**Style naming convention:** `RoleName / VariationName / WeightAlias` (separator configurable)

Example: `Heading / h1 / Bold`

### Font Weights
Global list of weights applied across all roles. Each weight has:
- `alias` — display name (e.g. "Bold")
- `value` — numeric CSS weight (e.g. 700)
- `figmaStyles` — per-font-slot Figma style name override (e.g. `{ primary: "Bold", secondary: "700" }`)

The `figmaStyles` per-slot lookup handles fonts that use non-standard style names in Figma (e.g. some fonts call Bold "700" or "ExtraBold").

### Font Slots
Up to N named font slots (primary, secondary, tertiary, etc.). Each slot maps to a font family + CSS fallback. Roles reference slots by name, not by family, so swapping a font family automatically updates all roles using that slot.

## Plugin Architecture (scripts.js)

No DOM access. All logic runs in Figma's JS sandbox. Communicates with the UI iframe via `figma.ui.postMessage` / `onmessage`.

### Message Protocol

**UI → Plugin:**

| `type` | payload | action |
|--------|---------|--------|
| `run-creater` | `{ state, scope: "all"\|"scale"\|"styles" }` | Main sync operation |
| `check-collections` | `{ scaleCollectionName }` | Check if collection already exists |
| `request-processed-data` | `{ state, exportType: "json"\|"css"\|"scss" }` | Generate export and trigger download |
| `resize` | `{ width, height }` | Resize plugin window |
| `cancel` | — | Cancel loading state |

**Plugin → UI:**

| `type` | payload | meaning |
|--------|---------|---------|
| `load-config` | `{ state }` | Config loaded from saved STRING variable on startup |
| `collection-check-result` | `{ existing: string[] }` | Names of collections that already exist |
| `finish` | `{ tally, errors }` | Sync complete |
| `error` | `{ message }` | Fatal error during sync |
| `processed-data-response` | `{ content, exportType }` | Export file content ready for download |

### Sync Sequence (run-creater)

1. `translateConfig(appState)` — normalise and validate state, run `migrateRoles` for backward compat
2. `typeMaker(config)` → `{ scaleSteps, roleStyles, errors }`
3. If `scope !== "styles"`: create/update FLOAT variables in the named collection
4. `await Promise.allSettled(fonts.map(f => figma.loadFontAsync(f)))` — load all required fonts; skip affected styles if a font fails
5. If `scope !== "scale"`: create/update text styles via `figma.createTextStyle()` / find existing by name
6. `style.setBoundVariable("fontSize", floatVar)` — bind size to the variable (try/catch — not all Figma plans support this)
7. Save config as STRING variable `__ttm316_config__`
8. `postMessage({ type: "finish", tally, errors })`

### Key Figma API Calls


### Export Formatters

All three live in `scripts.js` as pure functions: `ExportFormatter.toCSS`, `toSCSS`, `toJSON`.

**CSS** — generates `:root { --scale-N: Npx; --role-var-weight-size: var(--scale-N); ... }`
- Line height PERCENT 150 → `1.5` (multiplier)
- Letter spacing PERCENT -1 → `-0.01em`
- Letter spacing PIXELS 0.5 → `0.5px`

**SCSS** — generates `$scale-N` variables + `$type-scale` map + per-role maps + `@mixin apply-text-role`

**JSON** — full round-trip format: `{ meta, scale[], roles[], config }`

## UI Architecture (ui.html / Index.html + Scripts.js)

Single-file, no build step. Tailwind CDN via `<script src="https://cdn.tailwindcss.com">`. Google Fonts for Inter. All logic inline in `<script>`.

### State Management

**Plugin:** `appState` is a plain JS object. State is loaded from the Figma `__ttm316_config__` STRING variable on startup, and sent to `scripts.js` on every Run.

**Web App:** `appState` persisted to `localStorage` key `ttm316_state` with a 600ms debounce save (`scheduleSave()`). Loaded with `loadFromStorage()` at init. Reset via the Clear button restores `DEFAULT_STATE`.

### Render Strategy

Most renders are full re-renders of a DOM section. Exception: role card inputs use surgical DOM updates to preserve cursor position:

- **Role name change** → updates `.role-name-display` text only; no re-render
- **Short name change** → updates state only; no re-render
- **Unit select change** → toggles disabled/opacity on `.lh-value-input`, updates `.lh-unit-label` and `.ls-unit-label` text in the same `.variation-row`
- **Scale index change** → updates `.step-px-display` and `.lh-hint` text

Full re-render of one role card only: `rerenderRoleCard(roleIdx)` — used after add/delete variation. Preserves other cards' collapsed state.

Full re-render of all role cards: `renderRolesList()` — called when a non-surgical change occurs (e.g. font slot renamed, role added/deleted). Always preserves collapsed state before replacing innerHTML.

### Role Card Structure

```
role-card [data-idx]
  ├── Header (clickable → toggleRoleCard)
  │   ├── Chevron SVG
  │   ├── .role-name-display
  │   ├── variation count badge
  │   └── Delete role button
  └── .role-card-body (collapsible via CSS max-height)
      ├── Name input [data-rf="name"]
      ├── Short Name input [data-rf="shortName"]
      ├── Font Slot select [data-rf="fontSlot"]
      ├── Text Transform select [data-rf="textTransform"]
      └── VARIATIONS section
          ├── variation-row [data-vi=0]
          │   ├── Header: v1 badge · name input [data-vf="name"] · Step · index input [data-vf="scaleIndex"] · px display · delete
          │   └── Body: Line Height row · Letter Spacing row
          ├── variation-row [data-vi=1] …
          └── + Add Variation button
```

All `[data-rf]` inputs are wired by `wireRoleCard` → `handleRoleInput(ri, field, val)`.
All `[data-vf]` inputs are wired by `wireRoleCard` → `handleVariationInput(ri, vi, field, val, el)`.

### Schema Migration

Old configs (pre-variations refactor) used `variationCount`, `baseScaleIndex`, `scaleDirection`, `variationNames[]`, `variationOverrides[]`. `migrateRoles(roles)` converts these to the new `variations[]` format transparently. Called at:
- Plugin: `load-config` message handler, `btn-import-confirm` handler
- Web App: `loadFromStorage()`, `btn-import-confirm` handler

## Plugin UI Panels

### Scale Tab
Configure the type scale. Method buttons (Modular / Linear / Custom) show/hide the relevant inputs. Step naming scheme buttons apply immediately. The preview bar renders each step with live-preview text and an override input per step.

### Roles Tab
List of role cards. Each card is collapsible. Role-level fields (name, short name, font slot, text transform) apply globally to the role. Variation rows are per-variation — each has its own step index, line height, and letter spacing.

### Settings Sheet (bottom sheet, gear icon)
- **Basic:** System name, base font size (used for rem preview only, not scale math)
- **Font Families:** Slot name, family, fallback CSS. Slot names must match role `fontSlot` values.
- **Font Weights:** Per-weight alias, numeric value, per-slot Figma style name
- **Figma Output:** Collection name, style name separator, Skip Scale Variables toggle, Short Role Names toggle, style name preview

### Preview Sheet (bottom sheet, eye icon)
Two sub-tabs:
- **Scale** — each step rendered at its actual font size (capped at 52px for display) with primary font
- **Roles** — each role's variations rendered with resolved font, size, line height, letter spacing

### More Options Sheet (⋯ icon)
- Export CSS Variables → downloads `ttm316-tokens.css`
- Export SCSS → downloads `ttm316-tokens.scss`
- Save Config JSON → downloads `ttm316-config.json` (re-importable)
- Clear All → resets roles, scale, and font weights to defaults

### Run Dialog (Run button)
Pre-sync confirmation showing:
- **Scope** selector: All / Scale Vars only / Styles only
- **Summary** table: step count, role count, total style count
- **Font+Style Names** list: every `family + figmaStyle` pair that will be loaded — user verifies exact Figma names
- **Warning** if the scale collection already exists
- **Apply to Figma** confirm button

### Import
JSON import via file picker (import button) or drag-and-drop onto the plugin window. Both paths show a confirm overlay before overwriting state. Accepts both raw config objects and `{ config: ... }` wrapped exports.

## Key Algorithms

### `computeScale(scale, baseFontSize)`
Pure function. Returns `[{ index, name, px, rem }, ...]`. Used on every render in both plugin and web app. Handles `scaleOverrides` by patching the `px` array after generation.

### `typeMaker(config)`
Orchestrates scale generation + role style expansion. Returns `{ scaleSteps, roleStyles, errors }`. `roleStyles` is a flat array of one entry per role × variation × weight.

### `resolveRoleStyles(role, scaleSteps, config)`
Expands one role into style specs. Iterates `role.variations[]`, clamps `scaleIndex` to valid range, looks up font family from slot, and emits one style per font weight.

### `getFigmaStyle(weightDef, slotName)`
Priority: `weightDef.figmaStyles[slotName]` → `weightDef.figmaStyle` (legacy) → `weightDef.alias`.

### `migrateRoles(roles)`
Converts old `{ variationCount, baseScaleIndex, scaleDirection, variationNames, variationOverrides }` to new `{ variations: [{ name, scaleIndex, lineHeight, letterSpacing }] }`. Safe to call on already-migrated data (checks `role.variations` existence).

## Line Height Hint Ranges

| Font size   | Recommended multiplier |
|-------------|------------------------|
| < 16px      | 1.4–1.6 |
| 16–32px     | 1.2–1.4 |
| 32–64px     | 1.0–1.2 |
| > 64px      | 0.9–1.05 |

These hints appear next to each variation's line height input as a non-enforced guide.

## Known Limitations

1. **`setBoundVariable("fontSize", ...)`** — only works on Figma plans that support variable bindings. Fails silently (try/catch) and logs a notice.
2. **Font loading** — fonts must be installed in Figma's environment. A font that fails `figma.loadFontAsync` causes all styles using that font to be skipped. Reported in the finish tally errors.
3. **Line height variable binding** — `style.setBoundVariable("lineHeight", ...)` is not supported by the Figma API; line height is always set as a plain value.
4. **Figma sandbox** — `scripts.js` must use ES5-compatible syntax (no optional chaining `?.`, no arrow functions in some contexts, use `filter()[0]` instead of `find()`).

## Adding a New Feature: Checklist

1. **Add to `DEFAULT_STATE`** (web app) and `appState` (plugin) with a default value
2. **Add to `translateConfig`** in `scripts.js` so it survives the UI→Plugin round trip
3. **Add UI** in `ui.html` / `Index.html`
4. **Wire event listener** — scale inputs use `wireInput(id, field, parser)`; role fields use `data-rf`/`data-vf` attributes; settings use direct `addEventListener`
5. **Call appropriate render functions** after state mutation
6. **Add to export formatters** if the new property should appear in CSS/SCSS/JSON
7. **Add to `migrateRoles`** if it's a role-level property with a sensible default
